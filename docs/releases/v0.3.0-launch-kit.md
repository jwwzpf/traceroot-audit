# TraceRoot Audit v0.3.0 Launch Kit

## One-line positioning

TraceRoot Audit is an independent runtime audit layer for local agents: it watches live OpenClaw / MCP-style runtimes, alerts you when risky actions begin, and keeps a local audit trail you can review later.

## GitHub Release title

`TraceRoot Audit v0.3.0: runtime audit for local agents`

## GitHub Release body

```md
TraceRoot Audit v0.3.0 is the first release where the product is no longer just a scanner.

It now works as a runtime audit layer for local agents:

- scan local action surfaces before launch
- watch live OpenClaw / MCP-style runtimes while they run
- raise attention when risky actions begin
- keep a readable local audit trail you can review later

Start here:

```bash
npx traceroot-audit doctor --watch --host
npx traceroot-audit logs --today
```

What is new in v0.3.0:

- `doctor --watch --host` is now the main path for most users
- native OpenClaw / MCP runtime homes and event feeds are auto-discovered more reliably
- risky runtime actions can trigger live reminders while still being written into the local audit timeline
- `logs --today` now reads more like an audit report than a raw event dump
- launch-candidate acceptance coverage now verifies the core loop across 5 realistic runtime environments

TraceRoot Audit is not trying to replace sandboxing.
It sits beside the runtime as an independent audit layer:

- what is this agent trying to do right now?
- was it worth my attention?
- what exactly happened today?

That is the product.
```

## Screenshot ideas

Use these two screenshots for launch:

1. `doctor --watch --host`
   - shows TraceRoot has connected to the runtime
   - shows a live risky action reminder

2. `logs --today`
   - shows `今天的审计结论`
   - shows one clear risky action
   - shows where the action came from

## What not to say

Avoid claiming:

- full runtime isolation
- universal support for every agent platform
- blocking / killing all risky actions automatically

The strongest truthful claim is:

> agent 在跑，TraceRoot 也在跑；高风险动作一出现，用户立刻知道，之后还能回看完整轨迹。

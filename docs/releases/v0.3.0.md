# TraceRoot Audit v0.3.0

TraceRoot Audit v0.3.0 is the first release we would describe as a real runtime audit product, not just a scanner.

## What changed

- `doctor --watch --host` is now the main path for most users.
- TraceRoot can auto-discover common OpenClaw / MCP runtime homes and native event feeds.
- High-risk actions can trigger live reminders while still being written into the local audit timeline.
- `logs --today` now reads more like an audit report than a raw event dump.
- Launch-candidate acceptance coverage now verifies the core loop across 5 realistic runtime environments.

## The core loop

```bash
npx traceroot-audit doctor --watch --host
npx traceroot-audit logs --today
```

That loop is the product:

1. TraceRoot finds the runtime
2. TraceRoot watches live actions
3. risky actions trigger attention
4. the full trail stays available locally

## What this release is for

This release is meant for early market validation of a very specific promise:

> Your agent can run, and TraceRoot can run beside it as an independent audit layer.

It does not try to replace sandboxing or runtime isolation. It focuses on:

- independent runtime visibility
- readable local audit trails
- timely attention on risky actions
